# BeeCount-Platform 云协作封存状态（2026-02）

## 目标
- 作为 BeeCount App 实验态协作后端，保持接口稳定、可部署、可排障。

## 当前状态
- 认证、增量同步、实体读写、共享成员、设备会话、资料头像接口可用。
- `ALLOW_APP_RW_SCOPES` 默认开启，App 默认部署无需额外配置即可访问协作读写范围。
- Web 侧成员后台直管（按邮箱 add/update/remove）已具备。

## 本次封存重点
- 与 App 侧 identity 收敛策略对齐：协作账本按 `ledger_id(syncId)` 识别，不依赖同名猜测。
- 保持旧接口兼容，不做破坏性删除，便于后续重启。

## 已知限制
- 投影与历史脏数据场景仍需关注异常日志和审计信息。
- 协作链路仍属实验态，不承诺企业级治理能力（审计中心/审批流等）。

## 最小排障 SOP
1. 先确认 `ALLOW_APP_RW_SCOPES=true`，重启服务。
2. App 重登，确认 token/session 刷新。
3. 检查 `sync/push`、`write/*`、`read/*` 请求日志是否有 403/404/500。
4. 若为 identity 冲突，优先核对 ledger sync id 与成员关系，再做数据修复。

## 重启路线（后续）
1. 补齐协作主链路回归测试（包含多用户多设备并发）。
2. 增强投影幂等与错误可观测性。
3. 在稳定后再评估从实验态切换到默认稳定态。
